Navigation Hierarchy Explorations

Five different navigation approaches for the Circadian iPad app. Each balances different trade-offs: simplicity vs. discoverability, content space vs. wayfinding, and how prominently the primary action (Start Session) lives in the hierarchy.

Problems with current nav

  • "How it works" is onboarding — shouldn't be permanent nav
  • "About me" is vague — profile? preferences? medical info?
  • "Content selector" is a feature, not a destination
  • "Learn my face" is a one-time setup task in daily nav
  • No progress/data section — the most motivating part
  • "Start session" CTA has same visual weight as everything else
  • Flat hierarchy — everything at one level, no grouping

What the nav should do

  • Make "Start session" the most prominent thing on screen
  • Give progress/stats a dedicated home (encouragement)
  • Separate daily actions from setup/one-time tasks
  • Content should be accessible but not dominate nav
  • Help/onboarding should be contextual, not nav-level
  • Work for sick/tired patients — minimal cognitive load
  • Scale if features are added without getting cluttered
A Minimal Icon Rail — Maximum content space

Narrow icon-only sidebar keeps navigation accessible without eating content space. The Start Session button becomes a floating action button — always visible, always prominent, pulsing gently when it's time. Best for: maximizing the content area for data and media, keeping things clean for overwhelmed patients.

Proposed Information Architecture

Two approaches to organizing the same features — one lean (3 core destinations) and one with room to grow (5 destinations). Both remove one-time setup from primary nav and elevate progress tracking.

IA Option 1 — Lean (3 tabs)

Everything a patient needs in 3 taps. Setup and help are buried behind settings. Simplest possible for sick/tired/tech-averse patients.

  • 🏠 Home Primary
    • ☀️ Start Session CTA
    • 📊 Today's score + streak
    • 💡 Daily insight / recommendation
    • 📅 Treatment journey overview
  • 📊 Progress Primary
    • 📈 Session quality over time
    • 🗓 Session pattern / heatmap
    • 🏆 Milestones & achievements
    • 💤 Sleep patterns (if applicable)
  • 📺 Content Primary
    • 📺 Netflix
    • 📰 News feed
    • 🎵 Ambient sounds
    • 🧠 Trivia / activities
  • ⚙️ Settings Secondary
    • 👤 Profile / About me
    • 🤳 Face calibration Setup
    • 🔔 Notifications & reminders
    • ❓ How it works / FAQ
    • 🔗 Connected apps

IA Option 2 — Expanded (5 tabs)

More discoverable. Progress and Profile get their own tabs. Useful if the app evolves to include more social/personal features or deeper data views.

  • 🏠 Home Primary
    • ☀️ Start Session CTA
    • 📊 Today's snapshot
    • 💡 Daily insight / recommendation
  • 📊 Progress Primary
    • 🗺 Treatment journey map
    • 📈 Quality trends
    • 🗓 Session heatmap
    • 🏆 Achievements
  • 📺 Content Primary
    • 📺 Video (Netflix)
    • 📰 News & articles
    • 🎵 Audio & ambient
    • 🧠 Games & trivia
    • 🎯 Daily surprise
  • 👤 Profile Secondary
    • 📋 My treatment info
    • ⏰ My preferences & schedule
    • 🤳 Face calibration Setup
    • 🔗 Connected apps
  • ❓ Help Secondary
    • 📖 How light therapy works
    • 🔧 Troubleshooting
    • 💬 Contact support
    • 📋 FAQ
